# 基于 YOLO 与 ByteTrack 的眼斑双锯鱼跟踪项目

本项目构建了一个面向眼斑双锯鱼群体行为分析的计算机视觉流程，用于在顶视角视频中检测真实鱼体与鱼缸壁反射，并结合 ByteTrack 对个体进行多目标跟踪，为后续二维轨迹分析、行为指标计算、三维重建和 GCN 行为建模提供基础数据。

> [English]: [README.md](README.md)

## 项目概览

鱼缸顶视角视频中存在鱼缸壁反射、气泡遮挡、鱼体重叠和边缘漏检等问题。为降低这些干扰，本项目将检测任务设置为 `fish` 与 `reflection` 双类别，并在跟踪阶段仅对真实鱼体进行 ID 关联。

当前最佳候选方案为：

| 模块 | 当前方案 |
|---|---|
| 检测模型 | V5_train200_val100 |
| 跟踪器 | ByteTrack E2 |
| 检测类别 | fish / reflection |
| 跟踪类别 | 仅 fish |
| 当前用途 | 短时二维轨迹提取与行为分析预实验 |

## 项目流程

```text
视频输入
→ 视频预处理
→ fish / reflection 双类别检测
→ ByteTrack 多目标跟踪
→ 轨迹导出
→ 行为指标计算
→ 后续三维重建 / GCN 行为建模
```

## 项目亮点

- 使用 YOLO 构建 `fish` / `reflection` 双类别检测模型
- 将鱼缸壁上的完整鱼形反射单独作为 `reflection` 类别处理
- 使用 ByteTrack 对真实鱼体进行多目标跟踪
- 经过多轮模型迭代，从通用模型测试发展到 V5 检测模型
- 在同一验证集上，V5 的检测结果优于 V4
- 当前 V5 + ByteTrack E2 在 5 分钟顶视角测试片段中约观察到 3 次明显 ID 互换

## 检测类别与标注规则

| 类别 ID | 类别名 | 含义 |
|---:|---|---|
| 0 | fish | 真实鱼体 |
| 1 | reflection | 鱼缸壁上的完整鱼形反射 |

标注规则：

- 真实鱼体标注为 `fish`
- 鱼缸壁上的完整鱼形镜像标注为 `reflection`
- 普通光斑、水流纹理、气泡不标注
- 严重遮挡、轮廓不清或无法判断的鱼不标注
- 鱼体重叠但能区分个体时，分别标注为多个 `fish`
- 容易被模型误识别为真实鱼体的完整反射，应标注为 `reflection`

## 模型迭代记录

本项目已经完成多轮模型与数据集迭代：

| 阶段 | 主要内容 | 结论 |
|---|---|---|
| 通用 COCO 模型测试 | 直接使用通用 YOLO 模型推理 | 鱼体易被误识别为 COCO 类别，不能直接使用 |
| 早期 fish/reflection 基线 | 建立第一个双类别检测模型 | 可初步识别鱼体与反射，但漏检和 ID 碎片较多 |
| V2 | 增加训练轮数 | 单纯增加 epoch 不能解决困难场景问题 |
| V3 | 加入 TOP_test_2min 约 200 张人工标注图 | 检测完整性与跟踪稳定性提升 |
| V4 | 加入 TOP_test_5min 的 300 张人工修正样本 | 成为重要中间版本，并用于后续预标注 |
| V5 | 新增 train200 + val100，并在 V4 基础上微调 | 当前阶段最佳候选检测模型 |

## V4 与 V5 对比

为了避免不同验证集造成误判，后续使用同一套 V5 验证集对 V4 和 V5 进行公平比较。

| 模型 | 验证集 | fish AP | reflection AP | mAP@0.5 |
|---|---|---:|---:|---:|
| V4 | V5 val | 0.862 | 0.940 | 0.901 |
| V5 | V5 val | 0.903 | 0.952 | 0.927 |

结果表明，V5 在 `fish` 和 `reflection` 两个类别上均优于 V4，因此当前选择 V5 作为主要检测模型。

## 跟踪方案

当前跟踪方案为：

```text
V5 检测模型 + ByteTrack E2 跟踪配置
```

ByteTrack E2 是较保守的跟踪配置，其原则是：在不确定场景下宁愿断开 ID，也尽量避免将一条鱼错误连接到另一条鱼。

跟踪阶段只使用 `fish` 类别，不跟踪 `reflection` 类别。

阶段性结果：

- 在 5 分钟顶视角测试片段中，V5 + ByteTrack E2 约观察到 3 次明显 ID 互换
- 对于 3 条外观相似鱼、气泡遮挡、鱼体重叠和鱼缸壁反射场景，该结果已经具备短时二维轨迹提取的可行性
- 但当前流程仍不能认为已经完全解决 ID 保持问题

## 数据集构建概况

当前数据集由多轮抽帧、模型预标和人工修正构成：

| 批次 | 来源 | 用途 | 说明 |
|---|---|---|---|
| 早期基础数据 | TOP_final.mp4 均匀抽帧 | train / val | 建立初始 fish/reflection 数据集 |
| top2m | TOP_test_2min.mp4 | train | 约 200 张人工标注图 |
| top5m300 | TOP_test_5min.mp4 | train | 300 张模型预标 + CVAT 人工修正样本 |
| v5tr | TOP_final.mp4 20:00–25:00 | train | V5 新增 200 张训练图 |
| v5val | TOP_final.mp4 10:00–15:00 | val | V5 新增 100 张验证图 |
| v6tr | TOP_final.mp4 15:00–20:00 | planned train | 计划作为 V6 新增训练样本 |

注意：`TOP_test_5min` 中已有部分图像进入训练集，因此不能作为严格独立测试集，只能作为阶段性展示或验证视频。后续需要建立完全独立的 test set。

## 当前限制

当前项目仍处于预实验与流程验证阶段，主要限制包括：

- `TOP_test_5min` 并非严格独立测试集
- 严重气泡遮挡、鱼体重叠和边缘反射仍可能导致漏检或 ID 互换
- 偶尔可能出现反射或气泡被误检为 `fish` 的情况
- 当前结果适合短时二维轨迹提取和阶段性展示，但仍需要独立测试集验证泛化能力
- 后续需要对 ID 互换和遮挡片段进行人工复核，并标记 `identity_uncertain` 片段
