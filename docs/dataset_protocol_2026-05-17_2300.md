# 数据集构建与标注规则（截至 2026-05-17 23:00）

> 本文档记录截至 2026-05-17 23:00 的数据集构建过程与标注规范。  
> 该版本主要包含早期基础数据和 TOP_test_2min 新增样本，不包含 2026-05-18 之后的 V4/V5 数据扩充结论。

## 1. 数据来源

项目当前主要使用顶视角视频数据，核心视频包括：

| 视频 | 用途 |
|---|---|
| TOP.mp4 | 原始顶视角视频 |
| TOP_final.mp4 | 预处理后视频，用于抽帧和建模 |
| TOP_test_2min.mp4 | 2 分钟测试片段，用于补充训练样本和观察模型效果 |
| TOP_test_5min.mp4 | 后续阶段测试和困难样本扩充候选视频 |

当前阶段重点数据来自：

- TOP_final.mp4 的早期抽帧数据
- TOP_test_2min.mp4 中约 200 张人工标注图

## 2. 检测类别

本项目不是单纯检测鱼，而是同时检测真实鱼体和鱼缸壁反射。

类别设置如下：

| 类别 ID | 类别名 | 含义 |
|---:|---|---|
| 0 | fish | 真实鱼体 |
| 1 | reflection | 鱼缸壁上的完整鱼形反射 |

设置 reflection 类别的原因是：  
鱼缸壁上的完整反射很容易被模型误认为真实鱼体，如果不单独标注，后续检测和跟踪中会出现明显误检。

## 3. 标注规则

### 3.1 fish 标注规则

以下目标标注为 `fish`：

- 真实鱼体
- 轮廓清晰、可判断位置的鱼
- 即使与其他鱼接近，但仍能区分个体的鱼
- 重叠但仍可分开的多个鱼体

### 3.2 reflection 标注规则

以下目标标注为 `reflection`：

- 鱼缸壁上的完整鱼形镜像
- 形态接近真实鱼、容易被误检为鱼的反射
- 靠近鱼缸边缘的清晰反射鱼形

### 3.3 不标注的情况

以下情况不标注：

- 普通光斑
- 水流纹理
- 小气泡
- 模糊黑影
- 严重遮挡导致轮廓不清的鱼
- 只能凭经验猜测位置的鱼
- 不完整且不像鱼体的反射碎片

## 5. TOP_test_2min 样本抽取

TOP_test_2min 用于补充真实测试片段中的困难样本。

曾尝试较高频率抽帧，后续根据总帧数调整抽帧间隔，目标约为 200 张图像。

抽帧目标：

- 覆盖 2 分钟片段中的不同时间点
- 避免过密抽帧导致重复图过多
- 尽量保留反射、重叠、遮挡和边缘场景

该批次合并后使用 `top2m_` 前缀，加入训练集。

## 6. 数据集组织方式

YOLO 数据集建议采用以下结构：

```text
yolo_fish_reflection/
├── images/
│   ├── train/
│   └── val/
├── labels/
│   ├── train/
│   └── val/
└── fish_reflection.yaml
```

配置文件中类别应为：

```yaml
names:
  0: fish
  1: reflection
```

## 7. 当前数据集问题

当前数据集仍存在以下问题：

- 困难样本数量不足
- 反射、气泡、重叠场景仍需要补充
- 当前 train / val 划分还需要进一步规范
- 后续需要建立独立 test set
- TOP_test_5min 可以作为后续困难样本挖掘来源，但如果其中图像进入训练集，就不能再作为严格独立测试集

## 8. 下一步数据计划

下一阶段数据集建设重点：

- 从更长片段中抽取约 300 张困难样本。
- 使用当前较好模型进行预标注。
- 在 CVAT 中人工修正。
- 合并进训练集。
- 重新训练下一版模型。
- 建立严格独立的 test set，只用于最终评估，不参与训练和调参。

## 9. 数据建设原则

后续应遵循以下原则：

1. 不重复使用同一片段同时作为训练集和最终测试集。
2. 困难样本优先于随机样本。
3. 每次新增数据应记录来源视频、时间段、数量、用途和前缀。
4. 训练集、验证集和测试集应尽量分离。
5. 所有人工修正结果应保留 CVAT 导出文件，方便追溯。
