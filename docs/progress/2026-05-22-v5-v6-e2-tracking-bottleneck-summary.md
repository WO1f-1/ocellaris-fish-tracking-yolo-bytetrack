# 2026-05-22 项目进展：V5+E2 与 V6+E2 跟踪对比及瓶颈判断

> 项目：Ocellaris Fish Tracking with YOLO26s and ByteTrack  
> 当前阶段：V6+E2 跟踪验证完成，进入身份保持瓶颈分析阶段  
> 核心结论：V6+E2 在自动统计指标上明显优于 V5+E2，但人工复核仍发现 ID switch，说明当前瓶颈已由检测完整性转向身份保持与轨迹后处理。

---

## 1. 本次工作目标

本阶段主要验证 V6 检测模型是否能够在实际视频跟踪任务中替代 V5，重点比较：

- 每帧 fish 检测完整性；
- ByteTrack 生成的总 ID 数；
- 短碎片 ID 数；
- ID 生命周期；
- 人工复核 ID switch 次数。

测试视频为：

```text
TOP_test_5min.mp4
```

跟踪配置为：

```text
ByteTrack E2
```

对比模型为：

```text
V5+E2
V6+E2
```

---

## 2. 自动 tracking 指标对比

| 指标 | V5+E2 | V6+E2 | 变化 |
|---|---:|---:|---:|
| mean boxes/frame | 2.8822 | 2.9335 | +0.0513 |
| median boxes/frame | 3.0 | 3.0 | 0 |
| total IDs | 32 | 18 | -14 |
| short IDs < 90 | 16 | 4 | -12 |
| mean ID life | 1622.8 | 2938.2 | +1315.3 |

V6+E2 共处理 18030 帧，其中：

```text
fish 数量 = 3 的帧：16854
fish 数量 < 3 的帧：1158
fish 数量 > 3 的帧：18
```

V6+E2 的平均每帧 fish 数为 2.9335，中位数为 3，说明检测完整性已经非常接近真实鱼数 3。

---

## 3. 结果解读

从自动统计指标看，V6+E2 相比 V5+E2 有明显改进：

1. 平均每帧 fish 检测数更接近真实鱼数 3；
2. total IDs 从 32 降至 18，说明轨迹碎片化明显减少；
3. short IDs < 90 从 16 降至 4，短寿命碎片轨迹大幅减少；
4. mean ID life 从 1622.8 帧提升至 2938.2 帧，说明轨迹连续性增强。

这些结果表明，V6 检测模型为 ByteTrack 提供了更稳定、更完整的输入，尤其改善了漏检和轨迹断裂问题。

但人工复核发现，V6+E2 仍存在 4 次明显 ID switch，而 V5+E2 约为 3 次。因此，V6+E2 不能被表述为“完全优于 V5+E2”或“已解决身份保持问题”。

更准确的判断是：

```text
V6+E2 在检测完整性和轨迹连续性方面优于 V5+E2；
但在三鱼交叉、遮挡和近距离互动场景下，身份保持问题仍未解决。
```

---

## 4. 当前瓶颈判断

本项目当前已进入阶段性瓶颈。

瓶颈不再主要是：

```text
YOLO 检测能力不足
```

而是转移为：

```text
高相似个体在交叉、遮挡和近距离互动后的身份连续性维护问题
```

具体表现为：

- V6+E2 能减少漏检和短碎片 ID；
- 但 ByteTrack 仍可能在交叉后错误关联轨迹；
- 三条眼斑双锯鱼外观差异较小，外观 ReID 的预期收益有限；
- 后续优化重点应从“继续训练检测模型”转向“轨迹后处理与身份不确定片段标记”。

---

## 5. 是否继续训练 V7？

当前不建议立即随机扩充数据并训练 V7。

原因：

1. V6 检测性能已经较高；
2. V5/V6 检测指标差距较小；
3. V6+E2 的主要问题不是检测不到鱼，而是交叉场景下 ID 互换；
4. 随机抽帧继续训练可能无法解决 ID switch。

如果后续训练 V7，应采用 hard sample mining，而不是随机抽帧。

V7 数据应重点来自：

```text
missed_fish/
duplicate_fish/
fish_reflection_confusion/
edge_fish/
occlusion_crossing/
bubble_shadow_false_positive/
id_switch_segments/
count_not_3_frames/
```

V7 的目标不应是单纯提高 mAP，而应是：

```text
减少漏检
减少重复框
减少反射误检
降低 ByteTrack 输入噪声
提升 tracking 稳定性
```

---

## 6. 后续技术路线调整

由于三条眼斑双锯鱼外观相似，项目不再将 ReID 作为主要身份保持路线。

后续更合理的方向是：

```text
YOLO26s 检测
↓
ByteTrack 多目标跟踪
↓
轨迹 CSV 标准化
↓
identity_uncertain 片段标记
↓
tracklet stitching 轨迹拼接
↓
双视角三维轨迹重建
↓
运动连续性约束与社会身份规则判定
```

核心思想是：不依赖外观差异识别个体，而是利用时空连续性、三维位置、体型差异和社会互动方向维护身份。

---

## 7. 下一步计划

下一阶段重点不是继续盲目训练模型，而是建立轨迹质量控制体系：

1. 记录 V6+E2 中 4 次 ID switch 的帧号、时间点和发生场景；
2. 建立 `identity_uncertain` 标记规则，对交叉、遮挡和身份不确定片段进行标记；
3. 将 ByteTrack 输出统一整理为标准轨迹 CSV；
4. 计算速度、个体间距离、群体中心、MND 等基础行为指标；
5. 从 ID switch、漏检、重复框和反射混淆片段中构建 V7 hard sample 数据集；
6. 后续通过双视角同步采集和三维轨迹重建进一步提高身份连续性。

---

## 8. 阶段性结论

V6+E2 相比 V5+E2 显著改善了自动 tracking 指标，尤其体现在检测完整性、轨迹碎片减少和 ID 生命周期延长方面。因此，V6+E2 可作为下一阶段主要分析方案。

但人工复核显示 V6+E2 仍存在 4 次 ID switch，说明当前系统尚未完全解决身份保持问题。项目已从“检测模型优化阶段”进入“轨迹可靠性与身份连续性优化阶段”。后续工作应围绕 hard sample mining、identity_uncertain 标记、轨迹后处理和双视角三维轨迹重建展开。
