# 多目标跟踪流程记录（截至 2026-05-17 23:00）

> 本文档记录截至 2026-05-17 23:00 的鱼体多目标跟踪流程、观察指标和阶段性判断。  
> 该版本主要反映早期模型、V2/V3 阶段的跟踪测试思路，不包含 2026-05-18 之后最终确定的 V5 + E2 结论。

## 1. 跟踪目标

本项目的跟踪目标是对视频中的真实鱼体进行多目标跟踪，生成每条鱼随时间变化的二维轨迹。

检测阶段包含两个类别：

- fish
- reflection

但跟踪阶段原则上只跟踪：

- fish

也就是说，reflection 类别用于帮助检测模型学习区分真实鱼和鱼缸壁反射，但不作为后续行为分析的跟踪对象。

## 2. 使用方法

当前主要使用 Ultralytics YOLO 的 track 接口，并尝试 ByteTrack 作为多目标跟踪器。

典型跟踪流程如下：

1. 加载训练好的 fish/reflection 检测模型。
2. 输入测试视频。
3. 设置 tracker 配置。
4. 只保留 fish 类别进行跟踪。
5. 保存跟踪视频和 tracking labels。
6. 统计每帧检测数量、ID 数量和轨迹碎片情况。

## 3. 关键参数

跟踪阶段重点关注以下参数：

| 参数 | 含义 |
|---|---|
| conf | 检测置信度阈值 |
| iou | NMS 阈值 |
| imgsz | 推理输入尺寸 |
| tracker | 跟踪器配置文件 |
| classes | 指定需要跟踪的类别 |
| persist | 是否跨帧保持跟踪状态 |
| save_txt | 是否保存跟踪标签 |
| save_conf | 是否保存置信度 |

当前思路是降低检测置信度阈值，以减少漏检对跟踪的影响。  
但阈值过低也可能引入误检，因此需要结合实际视频观察和统计结果调参。

## 4. 输出格式

Ultralytics tracking labels 通常包含以下信息：

```text
class x_center y_center width height confidence track_id
```

字段含义：

| 字段 | 含义 |
|---|---|
| class | 类别 ID |
| x_center | 归一化中心点 x |
| y_center | 归一化中心点 y |
| width | 归一化框宽 |
| height | 归一化框高 |
| confidence | 检测置信度 |
| track_id | 跟踪 ID |

后续可将这些 label 转换为轨迹表：

| 字段 | 含义 |
|---|---|
| frame | 帧序号 |
| track_id | 个体 ID |
| x | 中心点 x |
| y | 中心点 y |
| w | 框宽 |
| h | 框高 |
| conf | 检测置信度 |

## 5. 跟踪评估指标

当前阶段主要使用以下指标评估跟踪效果：

### 5.1 每帧检测数量

真实鱼数约为 3 条，因此理想情况下每帧 fish 检测数量应接近 3。

重点观察：

- mean 是否接近 3
- median 是否为 3
- 是否经常少于 3
- 是否偶尔大于 3

如果经常少于 3，说明漏检较多。  
如果经常大于 3，可能存在误检。

### 5.2 总 ID 数量

同一段视频中，真实鱼只有约 3 条。  
如果总 ID 数量过多，通常说明轨迹频繁断裂或重新生成 ID。

但总 ID 数量不能单独判断跟踪好坏，因为过度强行关联也可能导致 ID 互换。

### 5.3 短碎片 ID 数量

短碎片 ID 指持续时间很短的 track id。  
短碎片越多，通常说明检测或跟踪不稳定。

可统计：

- 少于 30 帧的 ID 数量
- 少于 90 帧的 ID 数量

### 5.4 ID 互换

ID 互换是鱼体跟踪中最重要的问题之一。  
它通常发生在：

- 两条鱼靠近
- 鱼体交叉
- 短时间遮挡
- 气泡干扰
- 靠近鱼缸壁反射

ID 互换不能只靠统计表判断，必须结合视频人工检查。

## 6. 当前阶段观察

截至 2026-05-17 23:00，已经观察到：

- 早期模型可以跑通跟踪流程，但 ID 数量偏多。
- 增加 TOP_test_2min 样本后，检测完整性和 ID 稳定性有所改善。
- 检测模型质量直接影响跟踪结果。
- 反射误检、鱼体漏检和遮挡会引起 ID 断裂或 ID 互换。
- 后续需要在同一测试片段上比较不同模型的跟踪输出。

## 7. 后续跟踪改进方向

下一阶段应重点做：

1. 使用更好的检测模型减少漏检。
2. 针对反射、气泡和重叠场景补充训练样本。
3. 测试不同 ByteTrack 参数。
4. 比较不同参数下的总 ID 数量、短碎片数量和 ID 互换次数。
5. 对明显 ID 互换片段进行人工记录。
6. 后续导出轨迹 CSV，用于速度、加速度和个体间距离计算。

## 8. 行为分析准备

跟踪结果最终需要服务于行为分析。  
后续可从轨迹数据中计算：

- 鱼体二维位置
- 瞬时速度
- 加速度
- 个体间距离
- 最近邻距离
- 群体中心
- 离鱼缸边缘距离
- 活动热图
- ID 不确定片段标记

这些结果可进一步用于群体行为分析、三维重建和 GCN 图建模。
