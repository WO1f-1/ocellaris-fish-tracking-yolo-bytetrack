# 模型训练迭代记录（截至 2026-05-17 23:00）

> 本文档记录截至 2026-05-17 23:00 已经完成或能够合理总结的模型训练流程。  
> 不包含 2026-05-18 之后新增的 V4、V5 训练结论。

## 1. 通用 COCO 模型测试

### 目的

最初使用通用 YOLO 预训练模型测试视频目标检测流程是否能够跑通，包括：

- 视频读取
- GPU 推理
- 结果保存
- 多目标跟踪接口调用

### 观察结果

通用 COCO 模型能够在视频上产生检测结果，但并不能正确识别实验鱼体。鱼体可能被误识别为 COCO 数据集中的其他类别，例如 kite 等。

### 结论

通用 COCO 预训练模型不能直接用于眼斑双锯鱼检测。  
本项目需要训练自定义检测模型，并根据鱼缸场景设置专门类别。

因此后续检测任务改为：

| 类别 ID | 类别名 | 含义 |
|---:|---|---|
| 0 | fish | 真实鱼体 |
| 1 | reflection | 鱼缸壁上的完整鱼形反射 |

## 2. 早期 fish/reflection 双类别基线模型

### 目的

建立第一个自定义 fish/reflection 双类别检测模型，验证该建模方式是否可行。

### 数据基础

早期数据主要来自 TOP_final.mp4 的抽帧图像，并使用 CVAT 进行人工标注。  
标注类别为：

- fish
- reflection

### 训练设置

| 项目 | 设置 |
|---|---|
| 任务 | fish / reflection 双类别目标检测 |
| 模型 | YOLO26s |
| epochs | 约 50 |
| imgsz | 1024 |
| batch | 8 |
| device | GPU |

### 阶段性结果

该模型能够初步区分真实鱼体与鱼缸壁反射，但在实际视频跟踪中仍存在明显不足：

- 部分鱼体漏检
- 反射和真实鱼仍可能混淆
- 每帧检测数量不够稳定
- TOP_test_5min 中每帧平均检测数量低于真实鱼数 3
- 多目标跟踪中 ID 数量较多，说明轨迹碎片化比较明显

### 结论

早期基线模型证明了 fish/reflection 双类别路线是可行的，但模型尚不足以作为最终分析模型。  
后续需要通过更多困难样本改善漏检、误检和反射混淆问题。

## 3. V2：增加训练轮数的尝试

### 目的

V2 阶段尝试通过增加训练轮数提高模型性能，判断是否仅靠更长训练即可改善检测结果。

### 训练设置

| 项目 | 设置 |
|---|---|
| epochs | 200 |
| imgsz | 1024 |
| batch | 8 |
| patience | 40 |
| amp | False |

### 训练中遇到的问题

训练过程中出现过一些环境和依赖问题，包括：

- Ultralytics 字体文件下载卡住
- AMP 检查尝试下载模型文件
- 部分在线依赖下载受阻

后续处理方式包括：

- 使用本地权重路径
- 复制或补齐字体文件
- 设置 amp=False 避免相关检查问题

### 阶段性结果

V2 训练早停，约完成 76 个 epoch。  
最佳结果大约出现在 epoch 36 附近。

记录到的验证指标大致为：

| 指标 | 数值 |
|---|---:|
| all mAP@0.5 | 约 0.886 |
| all mAP@0.5:0.95 | 约 0.452 |

### 结论

单纯增加训练轮数并不能根本解决问题。  
模型的主要瓶颈不是训练轮数不足，而是困难样本数量和质量不足。

因此后续策略应从“继续加 epoch”转向“补充高质量样本”。

## 4. V3：加入 TOP_test_2min 样本

### 目的

V3 阶段重点补充真实测试片段中的人工标注样本，用于改善实际场景下的漏检和反射混淆。

### 新增数据

| 项目 | 内容 |
|---|---|
| 来源视频 | TOP_test_2min.mp4 |
| 数量 | 约 200 张 |
| 标注方式 | 人工标注 |
| 用途 | 加入训练集 |
| 文件前缀 | top2m_ |

### 数据意义

TOP_test_2min 样本相比早期均匀抽帧数据，更接近实际测试片段中的困难场景，包括：

- 鱼体靠近鱼缸壁
- 反射形态更明显
- 鱼体重叠或接近
- 小目标和边缘区域
- 气泡、水流纹理和反光干扰

### 阶段性表现

加入 TOP_test_2min 样本后，模型在实际视频上的表现较早期基线有改善：

- 检测完整性提高
- 每帧检测数量更接近真实鱼数
- 总 ID 数量下降
- 短碎片 ID 数量减少
- 跟踪结果更稳定

### 结论

V3 阶段说明：  
高质量、针对性强的人工标注样本，比单纯增加训练轮数更有效。

下一步应继续扩充更困难的样本，尤其是：

- 反射强烈区域
- 鱼体重叠场景
- 气泡遮挡场景
- 鱼缸边缘漏检场景
- 检测框不稳定或 ID 容易互换的片段

## 5. 截至 2026-05-17 23:00 的阶段判断

截至当前时间，模型迭代已经形成以下判断：

1. 通用 COCO 模型不适用于本项目。
2. fish/reflection 双类别检测是合理路线。
3. 早期基线模型可以跑通流程，但跟踪稳定性不足。
4. V2 说明单纯增加 epoch 不是主要解决方案。
5. V3 说明补充真实困难样本能显著改善检测与跟踪表现。
6. 下一阶段应继续使用模型预标注 + CVAT 人工修正扩充数据集。

## 6. 下一步计划

下一步建议：

- 从更长测试片段中抽取约 300 张困难样本。
- 用当前较好的模型进行预标注。
- 在 CVAT 中人工修正。
- 训练下一版模型。
- 在相同测试片段上比较不同模型的跟踪效果。
- 重点比较每帧检测数量、总 ID 数量、短碎片 ID 数量和明显 ID 互换次数。
